# SauceDemo E-commerce — Требования

## Обзор проекта (Project Overview)

SauceDemo — демонстрационное веб-приложение интернет-магазина, используемое для практики ручного тестирования.

Документ описывает наблюдаемые функциональные требования, наблюдаемые бизнес-правила, предположения QA и открытые вопросы, выявленные в ходе исследовательского тестирования приложения.

## Область тестирования (Scope)

В область тестирования входят следующие функциональные блоки:

- авторизация;
- каталог товаров;
- сортировка товаров;
- страница товара;
- корзина;
- оформление заказа;
- завершение заказа;
- выход из аккаунта.

## Вне области тестирования (Out of Scope)

В текущем проекте не проверяются:

- реальная обработка платежей;
- реальный расчёт стоимости доставки;
- регистрация пользователя;
- восстановление пароля;
- email-уведомления;
- прямая проверка backend и базы данных;
- кроссбраузерное тестирование за пределами выбранного окружения.

## Наблюдаемые функциональные требования (Observed Functional Requirements)

### FR-01 Login — Авторизация
- На странице авторизации отображаются поля `Username` и `Password`.
- На странице отображается кнопка `Login`, запускающая отправку формы авторизации.
- На странице указаны следующие тестовые пользователи:
  - `standard_user`
  - `locked_out_user`
  - `problem_user`
  - `performance_glitch_user`
  - `error_user`
  - `visual_user`
- Для указанных тестовых пользователей отображается общий пароль `secret_sauce`.
- При отправке формы с пустым полем `Username` отображается сообщение `Epic sadface: Username is required`.
- При отправке формы с пустым полем `Password` отображается сообщение `Epic sadface: Password is required`.
- При вводе неверного Username или Password отображается сообщение `Epic sadface: Username and password do not match any user in this service`.
- После неуспешной валидации рядом с полями Username и Password отображаются иконки ошибки.

### FR-02 Product Catalog — Каталог товаров

- После успешной авторизации пользователь перенаправляется на страницу каталога товаров.
- На странице отображается список из шести товаров.
- Каждый товар представлен отдельной карточкой.
- Карточка товара содержит изображение, название, описание, цену и кнопку `Add to cart`.
- На странице доступна сортировка товаров.
- В верхней части страницы отображается иконка корзины.
- В верхней части страницы доступна кнопка открытия бокового меню.

### FR-03 Product Sorting — Сортировка товаров

- В верхней части страницы каталога отображается выпадающий список сортировки товаров.
- По умолчанию в списке выбрано значение `Name (A to Z)`.
- В списке доступны четыре варианта:
  - `Name (A to Z)`;
  - `Name (Z to A)`;
  - `Price (low to high)`;
  - `Price (high to low)`.
- При выборе `Name (A to Z)` товары располагаются по названию в алфавитном порядке.
- При выборе `Name (Z to A)` товары располагаются по названию в обратном алфавитном порядке.
- При выборе `Price (low to high)` товары располагаются по цене от минимальной к максимальной.
- При выборе `Price (high to low)` товары располагаются по цене от максимальной к минимальной.
- После применения сортировки выбранное значение остаётся отображаться в выпадающем списке.

### FR-04 Product Details — Детали товара

- Пользователь может открыть страницу товара нажатием на название или изображение товара.
- На странице товара отображаются изображение, название, описание и цена.
- На странице товара доступна кнопка `Add to cart`.
- После нажатия `Add to cart` текст кнопки изменяется на `Remove`.
- После добавления товара рядом с иконкой корзины отображается счётчик.
- Пользователь может вернуться в каталог с помощью кнопки `Back to products`.

### FR-05 Shopping Cart — Корзина

- Пользователь может добавить товар в корзину со страницы каталога или со страницы товара.
- После добавления товара счётчик корзины увеличивается.
- Пользователь может открыть корзину через иконку в верхней части страницы.
- В корзине отображаются добавленные товары.
- Пользователь может удалить товар из корзины с помощью кнопки `Remove`.

### FR-06 Checkout Information — Информация для оформления заказа

- На странице `Checkout: Your Information` отображается форма с полями:
 - `First Name`
 - `Last Name`
 - `Zip/Postal Code`
- На странице отображается кнопка `Continue`.
- На странице отображается кнопка `Cancel`.
- При нажатии `Cancel` пользователь возвращается на страницу `Your Cart`.
- Если поля не заполнены появляется ошибка: `Error: First Name is required`
- Если поле First Name заполнено, а поле Last Name пустое, отображается сообщение `Error: Last Name is required`
- Если поля First Name и Last Name заполнены, а поле Zip/Postal Code пустое, отображается сообщение `Error: Postal Code is required`
- При заполнении всех обязательных полей и нажатии `Continue` пользователь переходит в `Checkout Overview`.

### FR-07 Checkout Overview — Обзор заказа

- На странице отображается заголовок колонки: `QTY`.
- Под заголовком `QTY` отображается количество единиц товара.
- На странице отображается заголовок колонки: `Description`.
- Под заголовком `Description` отображаются данные товара: название, описание и цена.
- При нажатии на название товара, можно перейти на страницу подробного описания товара.
- На странице отображается `Payment Information`: отображается способ оплаты.
- На странице отображается `Shipping Information`: отображается способ доставки.
- На странице отображается `Price Total` в который входят такие поля:
 - `Item total`: стоимость всех товаров.
 - `Tax`: сумма налога.
 - `Total`: итоговая стоимость заказа.
- На странице `Checkout Overview` есть кнопка `Finish` для перехода в раздел `Checkout: Complete!`
- На странице `Checkout Overview` есть кнопка `Cancel`, которая возвращает в раздел `Products`

### FR-08 Order Completion — Завершение заказа

- На странице отображается сообщение `Thank you for your order!`.
- Под сообщением отображается текст `Your order has been dispatched, and will arrive just as fast as the pony can get there!`.
- На странице отображается кнопка Back Home.
- При нажатии кнопки Back Home пользователь возвращается на страницу Products.
- счетчик корзины очищается.

### FR-09 Logout — Выход из аккаунта

- После открытия бокового меню отображается пункт `Logout`.
- При нажатии на пункт `Logout` пользователь выходит из аккаунта и перенаправляется на страницу авторизации.
- После выхода из аккаунта страница каталога товаров недоступна.
- При попытке вернуться назад с помощью кнопки браузера пользователь остаётся на странице авторизации.

## Наблюдаемые бизнес-правила (Observed Business Rules)

- Пользователь `locked_out_user` не может авторизоваться в приложении.
- Для всех указанных тестовых пользователей используется пароль `secret_sauce`.
- Для продолжения оформления заказа необходимо заполнить обязательные поля.

## Предположения QA (QA Assumptions)

- Текст сообщений об ошибках должен полностью отображаться внутри контейнера и не перекрываться другими UI-элементами.
- Иконки ошибок не должны перекрывать введённые значения в полях.
- Основные пользовательские сценарии должны оставаться доступными при изменении ширины окна браузера в пределах поддерживаемого разрешения.

## Технические наблюдения (Technical Observations)

- В ходе проверки через DevTools Network и прокси не обнаружены отдельные API-запросы для авторизации, добавления товаров в корзину и оформления заказа.
- Наблюдаемое поведение указывает на то, что основная логика демонстрационного приложения выполняется на стороне клиента.
- Прямая проверка backend, API и базы данных не проводилась, поскольку соответствующие интерфейсы и доступы отсутствуют.
