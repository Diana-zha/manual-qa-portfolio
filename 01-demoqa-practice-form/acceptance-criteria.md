# Acceptance Criteria

## Feature

DemoQA Student Registration Form

## Source

Критерии приёмки составлены на основе наблюдаемого поведения формы и предположений QA, так как официальные требования не предоставлены.

## Acceptance Criteria

AC-01: Пользователь может успешно отправить форму, если обязательные поля First Name, Last Name, Gender и Mobile заполнены валидными данными.

AC-02: Пользователь не может отправить форму, если поле First Name пустое.

AC-03: Пользователь не может отправить форму, если поле Last Name пустое.

AC-04: Пользователь не может отправить форму, если поле Gender не выбрано.

AC-05: Пользователь не может отправить форму, если поле Mobile пустое.

AC-06: Поле Mobile считается валидным, если содержит ровно 10 цифр.

AC-07: Пользователь не может отправить форму, если поле Mobile содержит менее или более 10 цифр.

AC-08: Если поле Email заполнено, система валидирует его формат.

AC-09: Email считается валидным, если содержит local-part, символ `@`, доменную часть и домен верхнего уровня, например `user@example.com`.

AC-10: Пользователь может выбрать Date of Birth через date picker.

AC-11: Пользователь может выбрать одно или несколько значений Hobbies с помощью чекбоксов.

AC-12: Пользователь может выбрать Subject из списка автодополнения.

AC-13: Пользователь может выбрать State и City из выпадающих списков, при этом список City зависит от выбранного State.

AC-14: После успешной отправки формы отображается модальное окно подтверждения.

AC-15: В модальном окне подтверждения отображаются данные, введённые или выбранные пользователем в форме.

AC-16: Пользователь может закрыть модальное окно подтверждения с помощью кнопки Close.
