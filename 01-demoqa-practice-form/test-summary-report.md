# Отчёт о тестировании

## Функциональность

DemoQA Student Registration Form

## Объект тестирования

DemoQA Practice Form

## Окружение

- OS: Windows
- Browser: Chrome
- URL: https://demoqa.com/automation-practice-form

## Объём тестирования

В рамках тестирования были проверены основные сценарии работы формы регистрации студента:

- успешная отправка формы с обязательными полями;
- успешная отправка формы со всеми полями;
- проверка обязательных полей;
- валидация поля Mobile;
- валидация поля Email;
- зависимость State и City;
- отображение данных в модальном окне подтверждения;
- закрытие модального окна подтверждения.

## Найденные дефекты

| Bug ID | Title | Severity | Priority | Status |
|---|---|---|---|---|
| BUG-01 | Модальное окно подтверждения не закрывается по кнопке Close | Medium | Medium | Open |

## Общий вывод

Основной функционал формы DemoQA Student Registration Form работает корректно: форма успешно отправляется при валидных данных, обязательные поля валидируются, поле Mobile ограничено 10 цифрами, невалидный Email блокирует отправку формы, зависимость State и City работает корректно.

Обнаружен дефект средней серьёзности: модальное окно подтверждения не закрывается по кнопке Close. При этом альтернативные способы закрытия модального окна доступны: клик вне области окна и клавиша Esc.

## Ограничения тестирования

- Тестирование проводилось только в браузере Chrome на Windows.
- Кроссбраузерное тестирование не выполнялось.
- Мобильная версия формы не тестировалась.
- Backend-валидация и сохранение данных на сервере не проверялись.
- Accessibility testing было выполнено только на базовом уровне.
